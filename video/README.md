# PathPilot AI Remotion Demo

A scene-based 1920×1080, 30 FPS product-demo project. It is isolated from the PathPilot frontend and backend and is designed for easy asset replacement.

## Important

The project has been scaffolded only. The final video has **not** been rendered. Dependencies have not been installed automatically.

## Composition

- Composition ID: `PathPilotDemo`
- Resolution: 1920×1080
- Frame rate: 30 FPS
- Current timeline: 12 scenes, 2 minutes 15 seconds (4,050 frames)
- Visual direction: modern startup presentation using navy, teal, white, soft gradients, and restrained motion

## Asset folders

All runtime media lives in the repository-level `video-assets/` directory. `remotion.config.ts` configures that directory as Remotion’s public folder.

```text
video-assets/
├── screenshots/  # PNG/JPG/WebP product screenshots
├── recordings/   # MP4/WebM UI recordings
├── audio/        # Background music and optional voice-over
└── branding/     # Logo and ending-card artwork
```

Missing visual assets show a branded placeholder with the expected path instead of crashing the composition. Missing background music is ignored.

## Scene architecture

Every scene is an independent component in `src/scenes/`:

```text
Scene01_Hook.tsx
Scene02_Landing.tsx
Scene03_CreateJourney.tsx
Scene04_Generation.tsx
Scene05_Roadmap.tsx
Scene06_Strategies.tsx
Scene07_ExplainWhy.tsx
Scene08_Replanning.tsx
Scene09_Dashboard.tsx
Scene10_Resources.tsx
Scene11_PDFShare.tsx
Scene12_Ending.tsx
```

`src/PathPilotDemo.tsx` controls order and duration. Durations are expressed in frames and can be changed independently.

## Reusable components

- `ZoomImage` — slow screenshot movement with missing-file fallback
- `RecordedClip` — source-time trimming, playback speed, and missing-file fallback for recordings
- `HighlightBox` — animated focus rectangle and label
- `ArrowCallout` — animated callout annotation
- `Subtitle` — frame-based subtitle cues
- `Transition` — consistent fade-and-rise scene entrance/exit
- `BackgroundMusic` — optional looped music with graceful failure
- `SceneLayout` — shared scene heading, background, and media frame

## Subtitles

Each scene supplies frame-based cues to `Subtitle`:

```tsx
<Subtitle cues={[
  {from: 8, to: 140, text: 'Your subtitle text.'},
]} />
```

Frames are relative to the start of that scene because each scene is inside a `Series.Sequence`. Keep subtitles concise and inside the safe lower-third area.

The current subtitle lines are placeholders. Draft voice-over slots and subtitle intent are documented in `NARRATION_AND_SUBTITLES.md`. Exact source-video feature boundaries and keep/remove decisions are documented in `RECORDING_EDIT_NOTES.md`.

## Local editing commands

When ready to work on the video:

```bash
cd video
npm install
npm run studio
```

The render command is included for later use, but do not run it until assets, timing, audio rights, and subtitles have been approved:

```bash
npm run render
```

## Asset replacement checklist

1. Place assets in the matching `video-assets/` subfolder.
2. Use the expected filename or update `src/assetManifest.ts`.
3. Prefer 1920×1080 recordings and screenshots at least 1600 px wide.
4. Use licensed or original background music only.
5. Keep voice-over separate from music if later added.
6. Preview every scene before rendering.
7. Confirm no localhost URL, API key, prompts, learner-sensitive data, or personal browser UI is visible.
